Инкрементальный алгоритм:

Алгоритм принимает множество S из n непересекающихся отрезков.
P - множесто концов отрезков S, является МТОП (никакие 3 точки не
лежат на одной прямой).

1. Возьмем любые два отрезка из n, построим их общую да-зону
	в виде 3-4 выпуклых многоульников, сохраним их.
2. Далее инкрементально добавляем отрезки, перестраивая да-зону:

		Добавляем k-тый отрезок:
	2.1. Строим некоторые (те же, что и в случае нет-зон) прямые, 
		соединяющие точки k-ого отрезка с точками прошлых k-1 отрезков.
		Таких прямых: 2 * (k-1).
	2.2. Для каждой да-зоны прошлого шага (их O(k)) строим РСДС (O(k^2)),
		содержащий сами выпуклые многоугольники + отрезки, порождаемые пересечением
		этой да-зоны с каждой из прямых шага 2.1.

			Для каждого полученного РСДС:
		2.2.1. Вычисляем статус аналогичный статусу из прошлого алгоритма в одной 
			из граней РСДС O(klog(k)).
		2.2.2. Обходим РСДС, проверяя осталась ли каждая из граней да-зоной.
		2.2.3. Выкидываем ненужные части, конструируем из РСДС новые выпуклые да-зоны.

O(1 + n * (n + n * (n^2) + nlog(n) + n + n)) = O(n^4)
